渗透测试基本流程和思路

一.确定目标

二. 信息收集（Reconnaissance）

被动收集

利用公开信息（如 WHOIS、DNS 记录、搜索引擎、GitHub、社交媒体）获取目标资产信息。

主动收集

1.主机发现

使用ping、arp-scan、nmap等探测内网或目标网段中存活主机

2.端口扫描

使用nmap、masscan识别开放端口及对应服务。

2.1基准扫描

3.Web 目录遍历

  • 使用dirsearch、gobuster、ffuf、dirb、gobuster、御剑等工具枚举敏感路径或文件

PHP探针

指纹识别

  • 识别操作系统、Web 服务器、CMS、框架版本（如 whatweb、Wappalyzer、nmap -sV）。

功能交互

  • 手动或自动化测试登录、上传、搜索等交互功能，挖掘潜在攻击面。

三. 漏洞验证和利用（Exploitation）

基于信息收集结果，结合CVE、POC或自定义脚本猜测可能存在的漏洞。

手工或使用msf验证常见漏洞类型

成功利用后获取初始访问权限（如webshell、低权限用户会话）。

口令爆破，当无其他漏洞时，尝试弱密码登录

四. 后渗透（Post-Exploitation）

从获得初始权限开始，所有扩大战果、深入控制的操作均属于此阶段。

利用反弹 Shell，将不稳定的WebShell权限转化为持久化的系统进程连接

1. 权限提升（Privilege Escalation）

  • 利用本地配置错误、SUID/能力滥用、内核漏洞或凭证复用，提权至 root/Administrator。

2. 本机信息收集

  • 枚举用户、计划任务、网络连接、SSH 密钥、浏览器历史、配置文件等，寻找横向移动线索。

3. 权限维持（Persistence）

  • 添加 SSH 公钥、创建计划任务、部署 Webshell 或 Meterpreter 后门，确保长期访问。

4. 内网渗透（Internal Network Penetration）

  • 代理隧道建立：通过 chisel、frp、proxychains 等工具将流量代理至内网。
  • 内网扫描：发现其他存活主机、开放服务
  • 横向移动：利用凭证传递（Pass-the-Hash/Ticket）、SMB 爆破、服务漏洞等跳转到其他主机。
  • 域渗透（如适用）：枚举域结构、利用 Kerberoasting、ACL 滥用等获取域控权限。

5. 痕迹清理

  • 清除日志、隐藏进程、绕过 EDR，降低被发现风险（通常在红队演练中更关注）。

五. 报告输出（Reporting）

  • 整理漏洞复现步骤、影响范围、风险等级（CVSS）。
  • 提供可落地的修复建议（如补丁、配置加固、权限最小化）。
  • 输出清晰的渗透路径图，体现攻击链完整性。
