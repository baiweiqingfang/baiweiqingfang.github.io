phar的攻击扩展面

phar文件由4个部分组成

(1)a stub:可以理解为一个标志,格式为xxx <? php xxx ;__ HALT_COMPILER() ;? >,前面内容不限,但必须 __ HALT_COMPILER() ;? >
来结尾,否则phar扩展将无法识别这个文件为phar文件。

(2) a manifest describing the contents:phar文件本质上是一种压缩文件,其中每个被压缩文件的权限、属性等信息都放在这部分。这
部分还会以序列化的形式存储用户自定义的meta-data,这是上述攻击手法最核心的地方。

(3)the file contents:被压缩文件的内容。

(4) [optional] a signature for verifying Phar integrity (phar file format only):签名,放在文件末尾

ps:php中,要生成Phar文件,要将php.ini中的phar.readonly选项设置为Off

可以和phar配合使用的方法

(1) include, include_once,require,require_once//文件包含函数

(2)file_get_contents,file_exsists,is_dir//文件判断,读取,操作函数

(3)XXE,和XXE中的文件读取相配合,达到phar的反序列化效果

(4)利用mysql的文件读取,从而达到phar的反序列化效果
