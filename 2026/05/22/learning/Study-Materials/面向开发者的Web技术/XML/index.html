1.XML

(eXtensible Markup Language)可扩展标记语言，用于传输和存储数据

优势：

(1)XML简化了数据的读取共享,XML以文本的形式进行储存,可以供不同的应用程序进行读取共享

(2)XML简化了数据的数据的传输,XML以文本的形式进行储存,可供不同的应用程序之间进行传输,读取

(3)XML简化平台的变更,XML数据以文本格式存储。这使得XML在不损失数据的情况下,更容易扩展或升级到新的操作系统、新的应
用程序或新的浏览器。

2.DTD

文档定义类型(document type definition，DTD)可定义合法的XML文档构建模块，它使用一系列合法的元素来定义文档的结构

DTD可被成行地声明于XML文档中，也可作为一个外部引用

2.1DTD构建模块

所有的XML均由以下构建模块构成：

2.2DTD元素

元素声明使用下面的语法：

声明只出现一次的元素

声明最少出现一次的元素

<! ELEMENT element-name(child-name+)>

声明出现零次或多次的元素

<! ELEMENT element-name(child-name*)>

声明出现零次或一次的元素

<! ELEMENT element-name(child-name?)>

属性值是必需的

<! ATTLIST element-name attribute-name attribute-type #REQUIRED>

属性不是必需的

<! ATTLIST element-name attribute-name attribute-type #IMPLIED>

属性值是固定的

<! ATTLIST element-name attribute-name attribute-type #FIXED value>

2.5DTD实体

实体是一种在XML文档中声明的命名引用，用于代替内容或标记。

实体引用是对实体的引用。

实体可在内部或外部进行声明。

当文档被XML解释器解析式，实体就会被展开

一个实体(参数实体除外)由三部分构成: 一个和号 (&), 一个实体名称, 以及一个分号 (;)
