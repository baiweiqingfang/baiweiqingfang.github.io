# XXE（XML Enternal Entity Injection，XML外部实体注入） XML解析器解析时未对XML外部实体加以限制，导致攻击者将恶意代码注入到XML中，导致服务器加载恶意的外部实体引发文件读取，SSRF，命令执行等危害操作。 防御方法: 禁用外部实体(PHP:可以将libxml_disable_entity_loader设置为TRUE来禁用外部实体) 其他的语言也可以使用这个方法进行防御。 ## 1.攻击流程 发现XML输入点 → 构造恶意XML（含外部实体） → 服务器解析XML → 触发外部实体加载（读取文件/访问内网） → 获取数据/探测信息 判断注入点： 1.查看网站源代码 2.查看HTTP的Request报文中是否出现Content-type:text/xml application/xml ## 2.利用方式 ### 有回显的XXE攻击 找到可能出现回显的位置，然后构造payload 利用xxe定义变量system去调用服务器支持的协议，利用file协议读取文件 ### 无回显的XXE攻击 使用BLind XXE漏洞构建一条外带数据(OOB)通道来读取数据 ## 3.JAVA中的XXE DocumentBuilder方法解析XML SAXbuilder方法解析XML SAXParerFactory ## 4.PHP中的XXE ### 4.1利用XXE读取文件名及权限内容 ### 4.2利用XXE使用外部实体读取文件 payload： 使用&name;读取文件内容 ### 4.3利用XXE命名实体及外部实体读取文件 ![image-20260402090346921](C:\Users\23214\AppData\Roaming\Typora\typora-user-images\image-20260402090346921.png) ### 4.4利用XXE进行SSRF漏洞利用 ### 4.5使用XXE利用php伪协议读取文件 某些敏感文件或PHP源码文件中经常包含特殊字符，如果直接读取XML 解析器会以为这些是XML的标签，导致解析失败报错 某些后端源码，直接读取的话会被服务器当成代码执行，导致只能看到执行结果，转成 Base64 后，就能拿到完整的、未执行的代码文本 ![image-20260521112621342](C:\Users\23214\AppData\Roaming\Typora\typora-user-images\image-20260521112621342.png) ### 4.6利用expect扩展进行命令执行 payload： 利用条件： **强依赖 PHP 环境：** `expect://` 协议是 PHP 特有的（严格来说是 PHP 的 PECL 扩展）。 **非默认自带扩展：** `expect` **不是** PHP 核心自带的扩展，必须由管理员手动通过 `pecl install expect` 安装，并在 `php.ini` 中明确启用。在实际业务中，极少有业务开发需要用到这个扩展。 **XXE 本身在被消亡：** 现代的 XML 解析器（如高版本的 libxml2）在默认情况下已经全面禁用了外部实体解析（DTD/External Entities）。现在能挖到一个普通的 XXE 已经不易，更遑论后面一连串的严苛条件。 ### 4.7无回显XXE带外数据 1.验证XXE漏洞是否存在 使用DNSLog或Collaborator 在kali开启http服务监听端口抓取HTTP请求GET链接内容 ![image-20260521221300775](C:\Users\23214\AppData\Roaming\Typora\typora-user-images\image-20260521221300775.png) 2.OOB（Out-of-Band，带外攻击） ![image-20260521224626609](C:\Users\23214\AppData\Roaming\Typora\typora-user-images\image-20260521224626609.png) 1.dtd文件内容： 第一步：使用file://伪协议读取文件内容 第二步：访问攻击者的服务器中的读取到的文件内容 由于攻击者的服务器中并没有此文件，虽然会返回404，但是会被记录在日志中，会将敏感数据带出来 ### 场景设定 -
