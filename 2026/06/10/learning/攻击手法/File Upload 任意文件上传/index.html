File Upload 任意文件上传

Web 应用开放了文件上传功能，对文件上传功能没有做限制或限制不足。

用户可以利用文件上传功能向服务器上传恶意文件，对服务器造成危害。

WebShell

WebShell 是一个网站的后门,也是一个命令解释器。通过web 方式,使用HTTP/HTTPS 协议传递命令消息到服务器,并且继承了web 用户的权限,在服务器上远程执行命令。WebShell 从本质上讲,是一类以网页脚本语言编写的代码执行环境，其核心功能是为攻击者提供对目标网站服务器的远程操作权限

WebShell 接收来自于Web 用户的命令,然后在服务器端执行,也称为网站木马、木马后门、网马等。

Web 容器 支持的脚本语言
Apache HTTPD PHP
IIS ASP、ASPX、PHP
Tomcat JSP、JSPX

  • 获取WebShell。

  • 控制网站。
  • 控制服务器。

漏洞利用

文件上传漏洞完美利用,受到以下条件限制:

· Web服务器开启文件上传功能,Web 用户可以使用该功能。

· Web用户({www|www-data|apache})对目标目录具有可写权限,甚至具有执行权限。一般情况下,Web 目录都有执行权限。
· 完美利用意味着文件可以执行,也就是说代码可以被服务器解析。服务器开启了PUT 方法。

双写绕过

二次渲染绕过：

其原理是通过Web服务器端的相关函数对用户上传的图片进行重新二次渲染生成新的突破，破坏掉原始图片文件中可能隐藏的恶意代码，从而有效防范了图片马等常见文件上传攻击手段

因此我们可以将一个正常显示的图片上传到服务器，寻找图片被渲染之后与原始图片部分对比仍然相同的数据块部分，将webshell代码插在该部分，进行上传

方法：

1.准备图片马

2.将原图片上传，下载渲染后的图片进行对比 ，可以利用工具找相同处，在相同的地方覆盖字符串，插入一句话木马，或者恶意指令

利用服务器特性进行绕过

Nginx:

1.Nginx文件名逻辑漏洞(CVE-2013-4547)
即在0.8.411.4.3/1.5.01.5.7版本中,如果上传一个1.gif (f后面有空格),
之后再访问http://ip:port/1.gif\x20\x00.php即可将1.gif解析为php(\x20为空格,\x00
为\0)。

2.Nginx配置不当解析漏洞
该漏洞与Nginx、php版本无关,属于用户配置不当造成的解析漏洞。

Apache：

1.Apache换行解析漏洞(CVE-2017-15715)，在2.4.0~2.4.29版本中,在解析PHP时,1.php\x0A将被按照PHP后缀进行解析。

2.Apache多后缀解析漏洞，在有多个后缀的情况下,只要文件名中含有.php后缀,即可被解析为php。

漏洞防御

前端（客户端）校验

使用JavaScript进行校验，一般值校验后缀名

后端（服务端）校验

1.主要通过$_FILE[‘xxx’[‘type’]]来获取content-type,之后进行校验

2.大多数通过getimagesize来校验文件头

3.文件后缀名校验

白名单和黑名单校验，使用白名单较为安全

代码角度：

  • 上传文件重命名，尽量少的从客户端获取信息，包括文件名、文件类型、文件内容等；
  • 文件内容检测；
  • 进行二次渲染，过滤掉图片马中的恶意代码；
  • 避免文件包含漏洞；
  • 严格处理文件路径，防御00 截断漏洞；
  • 检测Token 值，防止数据包重放。

业务角度：

强口令策略，避免恶意攻击者登录网站后台；

尽量避免Web 用户修改上传白名单。

Web 容器角度：

  • 及时更新Web 容器，防止解析漏洞产生。
  • 禁用Web 容器PUT 方法。

系统角度：

  • 避开空格、点.、::$DATA 等windows 系统特性。

服务器部署：

  • 严格控制权限，执行权限与写权限分离。
  • 建立单独的文件存储服务器，类似于站库分离。
