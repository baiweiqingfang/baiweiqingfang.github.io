4.1 Union 联合查询

order by 根据不存在的列排序会报错，可以用来判断原查询列数，为 union select 做准备。

适用于页面会回显数据库查询结果的情况。利用 union select 执行两条 select 语句，实现跨表查询。

必要条件：

  • 两条 select 语句查询结果具有相同列数。
  • 对应位置的数据类型兼容。
  • 页面存在可控回显位。

4.2 报错注入

当页面返回数据库错误信息时，可以人为构造类型转换错误，使错误信息携带查询结果。

优点：

  • 当 union 联合查询无法回显时可以尝试。
  • 对页面回显位要求较低。

缺点：

  • 生产环境大多会屏蔽详细报错。
  • 不同 SQL Server 版本和中间件返回信息不同。

4.6 宽字节、文件读写和系统命令差异

MySQL 常见的 load_file()、into outfile、secure_file_priv 不适用于 SQL Server。

SQL Server 中可能涉及以下高危功能：

  • xp_cmdshell
  • sp_OACreate
  • OPENROWSET
  • BULK INSERT

这些能力通常需要较高权限或特殊配置，安全测试时应重点检查是否被禁用、是否被最小权限限制，不建议在非授权环境尝试。

7. 防护建议

  1. 使用参数化查询或 ORM 参数绑定，禁止字符串拼接 SQL。

  2. 对数字、枚举、排序字段进行白名单校验。

  3. 数据库账号使用最小权限，不给业务账号 db_owner、sysadmin。

  4. 关闭详细错误回显，统一返回通用错误页面。

  5. 禁用高危扩展功能，例如 xp_cmdshell、OLE Automation。

  6. 对登录、搜索、详情页等入口增加安全测试用例。

  7. 使用 WAF、审计日志和异常 SQL 监控作为补充防线。
