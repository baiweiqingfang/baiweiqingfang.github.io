场景二：如果只有一个 IP，怎么做信息收集？

意味着你面对的是一台主机资产（可能承载着多个服务或网站）

拿到 IP（例如 202.x.x.x），核心思路是“横向摸排，服务攻坚”，主要围绕网络端口、操作系统、以及该 IP 承载的潜在 Web 站点展开。

1. 端口扫描与服务识别（最核心一步）

  • IP 只是个地址，真正干活的是里面的端口和服务。
    • 方法： 使用 Nmap 或 fscan 进行全端口（1-65535）扫描。
    • 命令示例： nmap -sS -sV -p- -T4 202.x.x.x (SYN 半开放扫描，探测服务版本)
    • 重点寻找高危端口：
      • 21/22/23/3389： FTP/SSH/Telnet/RDP（爆破、未授权、RCE 风险）。
      • 445/139： SMB 协议（内网或老旧 Windows 系统的永恒之蓝等）。
      • 1433/3306/5432/6379/9200： 数据库与缓存服务（MSSQL、MySQL、PostgreSQL、Redis 未授权、Elasticsearch，极易直接拿权限）。
      • 80/443/8080/7001/9000： Web 服务与常见中间件（Weblogic、Fastjson 等）。

2. 操作系统与硬件识别

  • 判断目标是 Windows 还是 Linux，这对后续制作 Payload（如木马、反弹 Shell 的命令）至关重要。
    • 方法： Nmap 的 -O 参数进行系统指纹探测。
    • 简单推断： 通过 Ping 值的 TTL（生存时间）粗略判断（Linux 默认通常为 64 左右，Windows 默认通常为 128 左右，受网络跳数影响会有所减少）。

3. IP 反查域名（寻找 Web 入口）

  • 有时候直接访问 http://IP 可能会显示 403 或 404，因为服务器配置了虚拟主机（Virtual Host），必须带域名访问。
    • 方法： 通过微步在线（ThreatBook）、Dnsdb、FOFA 输入 IP，查询该 IP 历史上解析过哪些域名。
    • 绑定 Host： 如果查到了域名，可以在本地修改 hosts 文件，将域名强制指向该 IP 再进行 Web 访问。

4. 旁站与 C 段资产探测（曲线救国）

  • 旁站（同 IP 站点）： 这台服务器上是不是同时跑着好几个网站？打不动主站，可以打同台机器上的另一个小网站，进而提权拿到整台服务器控制权。
  • C 段（同网段 IP）： 检查目标 IP 附近（如 202.x.x.1 到 202.x.x.254）有哪些机器。在红队演练或授权允许的情况下，如果目标 IP 密不透风，可以尝试攻破同网段内防御薄弱的兄弟机器，再通过内网横向移动（如 ARP 欺骗、内网扫描）打回目标机器。

5. 空间测绘历史缓存

  • 直接在 FOFA/Shodan 输入 ip="202.x.x.x"。
    • 好处： 空间测绘平台可能在几天前刚帮你扫过这个 IP，你可以直接看到它曾经开放过什么端口、网页长什么样，完全不需要直接给目标 IP 发包，实现完美的被动隐蔽收集。

🛠️ 总结与工具卡片

为了让你在脑子里更有画面感，我们可以把这两套流程提炼为两组核心工具链：

  • URL 收集链： Subfinder (子域名) $\rightarrow$ Wappalyzer (指纹) $\rightarrow$ Dirsearch (目录) $\rightarrow$ Google Hacking
  • IP 收集链： Nmap/fscan (端口/服务) $\rightarrow$ 微步反查 (域名) $\rightarrow$ FOFA/Shodan (历史暴露面)

无论哪种场景，信息收集的终极目标都是扩大资产接触面，直到发现那个最薄弱的服务或代码缺陷。
