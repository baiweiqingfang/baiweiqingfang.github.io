中间件

Web 服务器中间件

  • 主要漏洞：解析漏洞、目录遍历、配置错误。
  • 攻击特点：通常用来作为跳板，或者通过解析漏洞把一个 .jpg 当成 .php 执行。它们本身很少出现像 WebLogic 那样直接“白送”系统权限的反序列化漏洞。

Nginx / Apache

在 Web 服务器（如 Apache 或 Nginx）的配置中，有一个概念叫 Directory Index（目录索引）。

当你只输入 IP 而不输入具体文件名时，服务器会按照预设的优先级去文件夹里寻找“招牌”文件。通常的顺序是：

  1. index.html
  2. index.php (探针文件通常被重命名为此文件)
  3. default.php

应用服务器中间件

  • 主要漏洞：反序列化 (Deserialization)、后台弱口令上传 WAR 包。
  • 攻击特点：因为它们负责处理复杂的 Java 对象，所以逻辑更复杂，漏洞也更致命。一旦利用成功，通常直接就是 系统命令执行 (RCE)。
