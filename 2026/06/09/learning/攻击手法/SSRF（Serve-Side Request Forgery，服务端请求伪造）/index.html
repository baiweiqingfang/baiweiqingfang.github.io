SSRF（Serve-Side Request Forgery，服务端请求伪造）

由攻击者构造形成由服务器发起请求的一个安全漏洞，一般情况下，攻击的目标是从外网无法访问的内部系统（因为它是由服务端发起，所以能够请求到与它相连而与外网隔离的内部系统）

1.产生原因

很多服务器为了实现特定功能（如：抓取其他网页的标题、获取远程快照、下载文件、请求 API 等），必须具备访问其他机器的能力，且没有对目标地址做过滤与限制

2.漏洞危害

扫描内部网络，可以访问到本来不能访问的内网服务，或找出存活的内网主机或者开放端口

向内部主机发送精心构造的数据包以达到攻击的效果

DOS

进行暴力穷举，做进一步的信息收集

配合file，dict，gopher，ftp等协议进行请求，加以应用

3.漏洞防御

1.在服务端开启OpenSSL，按照规则生成数据，再用加密证书发送，经过校验才能通过

2.在服务端发起请求前开启认证cookie，password，token

3.正则防御

此类正则可以过滤掉内网的IP，从而达到防止内网SSRF的目的

4.禁用不必要的file，ditc，gopher等可能造成RCE或文件读写的协议（或只允许通过http的请求）

7.2302重定向绕过

针对SSRF服务器对用户输入的URL做过滤的情况

1.攻击者给SSRF服务器传一个看起来很正常的公网URL

2.SSRF服务器拿到这个 URL，一看是公网域名，不是127.0.0.1这种敏感地址，于是发起请求。

3.实际上看起来很正常的公网URL是攻击者控制的服务器。当它收到请求后，它不返回正常内容，而是返回一个302 Found响应，并在Location头部写上http://127.0.0.1:80

4.SSRF服务器在处理请求时，默认会遵循重定向逻辑，就会自动去访问Location里的地址。因为它已经过了第一步的地址检查关卡，所以它会直接请求自己本地的127.0.0.1

7.3DNS Rebinding Attack（DNS重绑定攻击）

攻击原理：利用服务器两次解析同一域名的短暂间隙,更换域名背后的ip达到突破同源策略或过waf进行ssrf的目的
