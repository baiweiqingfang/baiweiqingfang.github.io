免杀

2026.04.22 2 分钟阅读

免杀（Anti-Antivirus，简称AV Bypass）

通过各种技术手段，使恶意代码或渗透工具在目标系统中运行，而不被杀毒软件（AV）、终端检测与响应系统（EDR）或其它安全监测机制发现并拦截。

安全软件的三种检测方式

静态扫描：扫描文件，看它的特征码（Signature）是否匹配已知的病毒库。

启发式分析：根据代码中是否存在某些可疑的函数调用序列或逻辑结构，来判断它是否具有恶意倾向。

动态行为监控：代码运行后，如果它尝试注入进程、修改注册表核心项或加密大量文件，就会被立即掐断。

常见免杀方式

1. 静态免杀（针对静态扫描/指纹）

  • 特征码修改：通过改变二进制代码中的字符串、指令顺序或添加无用指令，避开杀毒软件的“黑名单”。
  • 加壳（Packing）：给程序穿一件“加密外套”，让安全软件扫描不到里面的真实代码。
  • 资源伪装：给木马加上正规公司的图标、数字签名或版本信息。

msfvenom

2. 内存免杀（针对“文件落地”）

  • 分离免杀：把恶意代码（Payload）和加载器（Loader）分开。程序运行后才去下载或读取 Payload。
  • 内存加载（无文件执行）：Payload 不保存在硬盘上，而是直接通过脚本（如 PowerShell）或者反射式加载技术直接塞进内存运行。

3. 行为免杀（针对动态监控）

  • 白名单利用 (LOLBins)：利用 Windows 自带的合法工具（比如 certutil, powershell, mshta）来运行恶意代码。因为这些工具是“名门正派”，不容易触发报警。
  • 系统调用 (Syscalls)：绕过 Windows 系统常见的监控接口，直接和底层内核对话，躲避安全软件的“钩子”。

4. 辅助手段

  • 延时执行：先睡眠（Sleep）几分钟再运行，以此骗过杀毒软件开头的快速沙箱扫描。
  • 反沙箱/反调试：程序运行前先检查自己是不是在虚拟机里，如果是，就装死不运行。
